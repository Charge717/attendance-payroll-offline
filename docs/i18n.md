# Internationalization Notes

## Current Policy

Core UI supports multiple languages, but payroll/legal calculation presets are Taiwan-focused by default.

## Supported Languages

- `zh-TW`: primary language
- `en`: core UI support

## Current English Coverage

The English locale currently covers:

- Main navigation and page-level labels
- Common form labels, buttons, tables, placeholders, and helper text
- Attendance preview labels and common payroll units such as NTD, minutes, hours, and days
- Payroll settlement HTML output
- Report center HTML output
- Printable payslip HTML output
- Spreadsheet workbook sheet names, column headers, and common text values
- Dynamic DOM output created after rendering through a MutationObserver translation pass
- Common browser `alert()` and `confirm()` messages

Some Taiwan-specific payroll/legal terms remain intentionally explicit because the calculation preset is still Taiwan-focused. Future work should extract locale dictionaries and payroll rule presets into separate files before adding more jurisdictions.

## Roadmap

Planned UI language expansion:

1. Traditional Chinese (`zh-TW`)
2. English (`en`)
3. Japanese (`ja`)
4. Korean (`ko`)
5. Spanish (`es`)

## Translation Boundaries

The interface can be translated, but payroll rules must remain explicit about their jurisdiction. For now:

- UI labels and project documentation can be localized.
- Common browser dialogs can be localized.
- Generated reports and exported workbook metadata must be localized when the active locale is not `zh-TW`.
- Calculation rules remain Taiwan-focused unless a separate rule preset is implemented.
- Any future country-specific preset must include tests and clear documentation.

## Contribution Rules

When adding or changing translations:

- Keep legal and payroll terms precise.
- Do not imply that Taiwan payroll presets apply globally.
- Add tests when a translation affects calculation labels, exports, or generated payroll reports.
