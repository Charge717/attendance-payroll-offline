# Internationalization Notes

## Current Policy

The project supports multilingual UI and documentation, but payroll and legal calculation rules are not automatically globalized.

The maintained calculation preset is Taiwan-focused. Translation work must not imply that the Taiwan preset applies to another jurisdiction.

## Supported Languages

- `zh-TW`: primary product language
- `en`: contributor-facing documentation and core UI support

## Translation Boundary

Safe to translate:

- Navigation labels
- Form labels and button text
- Help text and placeholders
- Documentation
- Browser alerts and confirmations
- Report and export labels, when they do not change calculation meaning

Requires extra review:

- Payroll, leave, attendance, overtime, insurance, pension, and deduction terms
- Generated payslip wording
- Spreadsheet column labels tied to legal meaning
- Any text that might imply compliance outside Taiwan

Not allowed as a translation-only change:

- Changing calculation behavior
- Renaming Taiwan-specific concepts into generic global concepts
- Adding another country or region without a separate rule preset proposal

## Current English Coverage

English coverage is intended to make the project understandable to contributors and evaluators. It does not mean the app is legally ready for non-Taiwan payroll use.

The English locale currently covers core UI, public project documentation, and common workflow labels. Taiwan-specific payroll terms should remain explicit and can be clarified through [glossary.zh-TW-en.md](glossary.zh-TW-en.md).

## Roadmap

Recommended order:

1. Keep `zh-TW` and `en` documentation accurate.
2. Extract locale dictionaries from the single HTML app.
3. Extract payroll calculation rules into named rule presets.
4. Add a `custom` preset for user-defined schedules and deductions.
5. Accept other language translations only when payroll-rule boundaries remain clear.

Potential future UI languages:

- Japanese (`ja`)
- Korean (`ko`)
- Spanish (`es`)

## Contribution Rules

When adding or changing translations:

- Keep legal and payroll terms precise.
- Use the glossary for Taiwan-specific terms.
- Do not imply that Taiwan payroll presets apply globally.
- Add tests when translation affects generated reports, exports, or payroll labels.
- Update [rule-presets.md](rule-presets.md) when proposing jurisdiction-specific behavior.
