# Internationalization Notes

## Current Policy

Core UI supports multiple languages, but payroll/legal calculation presets are Taiwan-focused by default.

## Supported Languages

- `zh-TW`: primary language
- `en`: initial English support

## Roadmap

Planned UI language expansion:

1. Traditional Chinese (`zh-TW`)
2. English (`en`)
3. Japanese (`ja`)
4. Korean (`ko`)
5. Spanish (`es`)

## Translation Boundaries

The interface can be translated, but payroll rules must remain explicit about their jurisdiction. For now:

- UI labels and project documentation can be localized.
- Calculation rules remain Taiwan-focused unless a separate rule preset is implemented.
- Any future country-specific preset must include tests and clear documentation.

## Contribution Rules

When adding or changing translations:

- Keep legal and payroll terms precise.
- Do not imply that Taiwan payroll presets apply globally.
- Add tests when a translation affects calculation labels, exports, or generated payroll reports.
